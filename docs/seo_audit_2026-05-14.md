## TL;DR

The site has zero discoverable SEO infrastructure. There is **no `sitemap.ts`, no `robots.ts`, no `llms.txt`, no structured data (JSON-LD), no real OG image, no canonical strategy beyond the root layout, and no per-page metadata on `/about`, `/projects`, `/books`, `/writing`.** Image optimization is globally disabled in `next.config.mjs` (`images.unoptimized: true`), which is the single biggest CWV risk. Most secondary pages are `"use client"` (no SSR-friendly metadata).

The good news: the **entity moat is the play** — you already have real press hits (Canal 10, Perfil, Cba24n, Iproup, Infonegocios, etc.), real GitHub presence, Tegu (the company you're co-founding) is itself becoming searchable. The Person + Organization + sameAs graph, plus a content surface that lets LLMs cite *"according to Matías Vallejos…"*, is what compounds. Everything below is in service of that.

## AEO (Answer Engine Optimization) posture

**Where you can win:**

- **Perplexity** — Reddit + recency-weighted. You're active on X but not Reddit (`r/argentina`, `r/devsarg`). A monthly post in `r/devsarg` discussing product engineering, Tegu's stack, LatAm builder lessons would compound. Founder presence on Reddit is the single highest-leverage AEO action for a Spanish-speaking founder in 2026.
- **ChatGPT / SearchGPT** — Bing index–driven. **Register at Bing Webmaster Tools** (15 minutes); submit sitemap. ChatGPT cites Wikipedia 47.9% of the time at the top — the long game is a Wikipedia entry for Tegu (eventually) anchored by your existing press footprint.
- **Claude (with search)** — conservative, formal-source-weighted. The Person schema + press entries + Tegu Organization graph is what gets cited.
- **Google AI Overviews** — query fan-out + passage retrieval. Blog posts with question-format H2s ("¿Cómo migrar un Identity Server con AI?", "¿Qué es Flipped Interaction?") win passage retrieval.

**Pragmatic AEO checklist:**

1. Ship `llms.txt` (curated ~25 links: home, about, blog index, top 5 blog posts, projects, press) + `llms-full.txt` (concatenated markdown of the above). Defensive only — do not invest more than 4 hours.
2. Allow AI crawlers in `robots.txt`.
3. Add `Person` + `Organization` schema with `sameAs` so the entity graph resolves consistently.
4. Question-format H2 headings on every blog post (translate existing posts to start with `¿…?`).
5. Visible "Actualizado: mayo 2026" date.
6. Author bio block at the end of every post.
7. DIY monitoring: weekly script (`cron` + Anthropic + OpenAI APIs) running 20 es-AR prompts ("quién es matías vallejos", "fundador de tegu", "product engineer argentina córdoba", "co-founder docta valley") — log citation rate per platform. ~$10/mo in API spend.

## Performance (CWV) — code-only posture

Without CrUX field data the best signal is the build. Observed risks:

- `images.unoptimized: true` → no AVIF/WebP, no responsive `srcset`. Estimated LCP penalty: 1–2s on mobile.
- Hero `<img src="/images/me.png">` is the LCP element and is not `priority`/`fetchpriority="high"`. Estimated LCP: ~3s mobile.
- Framer Motion is imported on most pages — JS payload on first paint. Mitigation: keep motion in `"use client"` leaf components only; ensure the static shell renders without it.
- Three render-blocking concerns to verify post-deploy: font loading (no `next/font` usage observed), Vercel Analytics + Speed Insights (wired? CLAUDE.md says yes), `framer-motion` initial bundle.

**Action:** ship Lighthouse CI in GitHub Actions, run on PR previews, gate merges on `LCP < 2.5s` mobile + `INP < 200ms`.

## What NOT to do

1. **Don't build programmatic location/topic pages without unique data** — same August 2025 SpamBrain trap that applies to Tegu applies here. A personal site doesn't need them anyway.
2. **Don't buy backlinks.** The press footprint is enough; cheap link farms are detected and would actively hurt.
3. **Don't AI-generate blog content at scale.** QRG September 2025 update flags scaled AI content as Lowest quality. Hand-write or human-curate every post.
4. **Don't ship a massive `llms-full.txt`** dumping every page — curate to ~20-40 highest-value links.
5. **Don't over-engineer hreflang for a single market.** You're `es-AR` and `en` for international Tegu/personal-brand discovery; that's it. Skip `es-MX`/`es-CL` until you actually expand.
6. **Don't add a third-party chat widget / cookie banner** unless legally required — every script is INP risk.
7. **Don't keep `<img>` for hero / blog post images** — `next/image` (with optimizer re-enabled) is non-negotiable for CWV.

## Caveats

- This audit is **code-only**. Without GSC impressions/clicks, Bing data, CrUX field data, or GA4 traffic, scoring of CTR / ranking / CWV is a posture estimate based on what the source ships. After registering GSC + Bing (Task #9), re-baseline in 30 days.
- The site is **pre-traffic** — most of the SEO score is about getting indexable signals correct, not optimizing existing rankings. The pattern is: ship the infrastructure → register the consoles → wait 30 days → measure → iterate.
- **Health Score 38/100 is a launch-state number**, not a quality judgment. With the Week 1–2 tasks shipped it jumps to ~65; full 90-day plan delivered cleanly takes it to 80+.
- **The biggest single lever is the Person entity graph.** Everything else (sitemap, CWV, content) is leverage on top of that. If you only had time for one thing it would be: ship Person + Organization + WebSite JSON-LD with sameAs to every social/Tegu/press URL, with stable `@id` references across pages.
- Press list in `src/data/press.ts` may already be slightly out of date relative to recent hits — refresh it whenever a new outlet publishes so the schema stays accurate (LLMs and Google both punish stale entity data).
