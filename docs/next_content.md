# Next content — viral content playbook

> Site is at top-tier design. From here, the multiplier is content cadence + viral mechanics. This document captures the framework for posts that compound (X, IG, blog).

---

## STEPPS framework (Berger — *Contagious*)

To be shareable — not just liked — content needs ≥3 of 6:

| Dimension | Check question | Founder-técnico application |
|---|---|---|
| **Social currency** | Does it make the sharer look smart? | Counter-intuitive insights, fresh data, build-in-public numbers nobody publishes |
| **Triggers** | Is there a daily-life cue? | Associate posts with recurring moments (Mon AM, end of month, after a ship) |
| **Emotion** | High-arousal feeling? | Take a position. Lukewarm doesn't viralize. Indignation, awe, amusement, anxiety. |
| **Public** | Is adoption observable? | Frameworks with names. Repos people fork. Quotes worth screenshotting. |
| **Practical value** | Useful TODAY? | Tutorials with concrete actionable. Repos that solve real pain. |
| **Stories** | Real conflict + resolution? | Postmortems, failure stories, "I lost X to learn Y" |

**Critical non-obvious insight (per the BE doc)**: high-arousal emotions (awe, indignation, amusement, anxiety) get shared. Low-arousal emotions (sadness, satisfaction, complacency) don't. Sad threads rarely viralize. Indignant threads do. **Choose carefully** — anger viralizes but degrades long-term brand.

## SUCCESs framework (Heath — *Made to Stick*)

To be remembered:

- **Simple** — one core idea per post. If it can't be tweet-sized, it's two posts.
- **Unexpected** — violate the reader's expectation in the first line.
- **Concrete** — use real names, real numbers, real dates. "Cliente X" beats "a customer".
- **Credible** — show, don't claim. Numbers > adjectives.
- **Emotional** — hook into identity, fear, awe, or pride.
- **Stories** — protagonist + conflict + resolution. Lists die, stories travel.

**Curse of Knowledge warning** (from the doc): once you know something, you can't imagine not knowing it. Default check before publishing: would your mom understand the hook? If not, you're in curse of knowledge.

## Hook templates (high-conversion for X/threads)

1. **Identity → outcome reframe**
   > "Yo era X. Ahora soy Y. Esto es lo que aprendí."

2. **Most → few**
   > "La mayoría hace X. Los que ganan hacen Y."

3. **Specific cost recall**
   > "Tres errores que cometí cuando empecé. El tercero me costó USD Y."

4. **Real cost > apparent cost**
   > "X cuesta Y. Pero el costo real es Z."

5. **Curiosity gap**
   > "Nadie habla de esto: [observación específica]."

**Closing pattern (peak-end rule)**:

- Synthesis in 1 line
- One micro-action for today (Fogg ability)
- Optional: "If this helped, retweet" — commitment + social proof loop

**What does NOT work in X 2025-2026** (per doc):

- "Thread 🧵" as opener (dead since 2023)
- Generic top-10 lists
- Obvious engagement bait

## Founder-técnico content angles (Playbook E)

These are the specific high-leverage formats for Mati's positioning:

1. **Postmortem público** — "Perdimos USD X por este bug. Acá está lo que aprendimos."
2. **Contrarian framework** — "Todos dicen X. Mi data dice Y."
3. **Behind-the-scenes con métricas** — "Cómo conseguí los primeros 100 clientes — con los números completos."
4. **Lección anti-gurú** — "Probé el playbook de [gurú famoso]. Estos son los resultados reales."
5. **Build-in-public weekly** — "Tegu Weekly: 65 nuevos usuarios, 8 pros pagaron, 1 churn."
6. **Build log mensual** — "Lo que cambió en Tegu este mes (con los números)."
7. **Shipped & failed** — "Lancé X. Funcionó / no funcionó. Acá está el por qué."

**What does NOT viralize for this segment**:

- "10 lessons I learned in 10 years" — saturated
- Inspirational quotes without context
- Wins without numbers
- POV-less content

## Cadence (90-day commitment)

**Días 1-30 — establecer presencia**

- Bio: anchor con credencial técnica + lo que estás construyendo
- Pinned tweet: output más impresionante (no manifesto)
- 5 posts/semana X, 3/semana IG. Mere exposure + Fogg ability
- 80% give, 20% ask

**Días 31-60 — narrativa transportable**

- Build-in-public semanal con métricas reales
- Cada post = caso concreto
- Engagement con 5-10 founders más grandes que vos por día

**Días 61-90 — Unity y diferenciación**

- Identificá tu "Purple Cow" (en este sitio = ShipStatus floating + build logs interactivos)
- Lanzá algo open-source o gratis (reciprocity a escala) → ya vienen haciendo esto con repos
- Empezá a aparecer en podcasts hispanohablantes

## Operational checklist before publishing

Before posting any thread / blog / X post, run this:

- [ ] Does it pass ≥3 STEPPS dimensions?
- [ ] Is the hook concrete (numbers, names, dates) — not abstract?
- [ ] Does it have a CLOSE that's memorable (peak-end)?
- [ ] Is there a micro-action the reader can take TODAY?
- [ ] Did I check curse of knowledge — would a non-expert understand the first line?
- [ ] Is the emotion high-arousal (or am I sharing into the void)?

If 5/6 pass, ship. If less, rewrite.

## What feeds the site

The site doesn't generate content on its own. It needs to be fed:

| Site element | Fed by |
|---|---|
| Build logs (Now section) | Monthly build retrospective |
| Tegu metrics (Hero Results card) | Weekly metric snapshot |
| Ship gallery (Floating ShipStatus) | Per-release event |
| Latest blog (Writing section) | Monthly long-form post |
| Press marquee | Per-press-feature event |
| Open Source | Per-new-repo event |

**The site is a dashboard. Content is the underlying API. If the API stops updating, the dashboard goes stale.**
